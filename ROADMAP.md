# Local Wiki 구현 로드맵

집 서버에서 구동되며 마크다운 파일을 기반으로 실시간 업데이트를 지원하는 로컬 위키 시스템 구현 계획입니다.

## 🎯 목표
- **파일 기반 저장소**: `.md` 파일을 직접 수정하여 위키 내용을 관리.
- **마크다운 렌더링**: 웹 브라우저에서 마크다운 문법이 적용된 깔끔한 문서 확인.
- **실시간 반영**: 서버의 파일 변경을 감지하여 웹 페이지에 즉시 반영.
- **심플한 UI**: 문서 간 이동이 편리한 내비게이션 및 가독성 높은 디자인.

## 🛠 기술 스택 (제안)
- **Backend**: Node.js + Express (파일 시스템 접근 및 API 제공)
- **Frontend**: React 또는 단순 HTML/JS + Tailwind CSS
- **Markdown Parser**: `marked` 또는 `markdown-it`
- **Real-time**: `chokidar` (파일 감지) + `Socket.io` (클라이언트 알림) 또는 SSE

## 🗺 단계별 구현 계획

### Phase 1: 기본 구조 및 파일 시스템 설정
- [ ] 프로젝트 디렉토리 구조 설계 (`/data` 디렉토리를 위키 저장소로 사용)
- [ ] 기본 서버 설정 (Express)
- [ ] 파일 목록 읽기 API 구현 (위키 페이지 리스트 반환)
- [ ] 특정 파일 내용 읽기 API 구현 (마크다운 텍스트 반환)

### Phase 2: 기본 웹 인터페이스 구현
- [ ] 마크다운 렌더러 통합 (Client-side 또는 Server-side rendering)
- [ ] 사이드바 내비게이션 구현 (파일 목록 기반)
- [ ] 문서 보기 영역 구현 (마크다운 렌더링 결과 출력)
- [ ] 기본 CSS 스타일링 (Typography 및 레이아웃)

### Phase 3: 실시간 업데이트 기능 구현
- [ ] `chokidar`를 이용한 `/data` 폴더 파일 변경 감지 로직 구현
- [ ] WebSocket (`Socket.io`) 또는 SSE를 통한 변경 알림 시스템 구축
- [ ] 클라이언트에서 알림 수신 시 콘텐츠 자동 갱신 기능 구현

### Phase 4: 위키 편의 기능 추가
- [ ] **Wiki-links**: `[[문서명]]` 형태의 내부 링크 지원
- [ ] **검색 기능**: 파일 내용 기반의 간단한 키워드 검색
- [ ] **코드 하이라이팅**: `prism.js` 또는 `highlight.js` 적용
- [ ] **자동 인덱싱**: 메인 페이지(`index.md`) 자동 생성 및 렌더링

### Phase 5: 배포 및 최적화
- [ ] Dockerfile 작성 (집 서버 배포용 컨테이너화)
- [ ] `docker-compose.yml` 설정 (데이터 볼륨 마운트)
- [ ] 성능 최적화 및 버그 수정
